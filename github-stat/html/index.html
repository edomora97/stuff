Qual'è il miglior stile di programmazione?

Una semplice analisi per ricercare lo stile di programmazione più usato nei progetti opensource.

I dati si riferiscono ai soli repository pubblicati su GitHub, filtrati secondo alcuni criteri, e analizzati tramite un software di indagine statistica.

Sei pronto per iniziare?

Via!

I dati

In questa indagine sono stati analizzati oltre 32 GigaByte di codici provenineti da circa 7000 repository pubblici su GitHub.

In totale sono state analizzate oltre 900 milioni di righe di codice, in oltre 3 milioni di file!

Sono stati selezionati i 1000 repository più emergenti dei seguenti linguaggi: C, C++, PHP, JavaScript, C#, Python, Ruby.

Avanti

L'indagine

Tutti questi dati sono stati sottoposti ad un'accurata analisi per determinare lo stile di programmazione più usato, ma non solo!

L'obiettivo principale di questa statistica è rispondere ad alcuni dei dibattiti più accesi tra i programmatori:

  • La posizione delle parentesi graffe
  • I caratteri di indentazione
  • Gli spazi nelle parentesi tonde
Avanti

Parentesi Graffe

Esistono molti modi di mettere le parentesi graffe in un programma, ecco quelli analizzati:

In fondo alla riga A capo Tutto in una riga
int main() {
    return 0;
} 		int main()
{
    return 0;
} 		int main() { return 0; }

Caratteri di tabulazione

Molte discussioni nascono anche dai caratteri scelti per indentare il codice. Non esiste uno standard, quindi molti hanno idee contrastanti se è meglio indentare con TAB o con spazi semplici.

Spazi nelle parentesi tonde

Mettere degli spazi all'interno delle parentesi tonde non è obbligatorio, alcuni programmatori le mettono, altri no. Ecco a cosa si riferisce l'indagine:

Senza spazi nelle parentesi Con gli spazi nelle parentesi
if (i >= 0) if ( i >= 0 )

I vincitori

A conti fatti i vincitori di questa analisi sono:

Le parentesi graffe a fine riga talvolta di molto, talvolta di poco

L'indentazione usando gli spazi veramente una brutta cosa...

Le parentesi tonde senza spazi vittoria schiacciante!
